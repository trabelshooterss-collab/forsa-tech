## 1. تحليل دوبيزل بالتفصيل

### 1.1 الهيكل العام للموقع

#### 1.1.1 الصفحة الرئيسية
**العناصر الرئيسية:**
- **شريط التنقل العلوي (Header)**
  - شعار الموقع في المنتصف
  - زر "نشر إعلان" بارز
  - أيقونات: الإشعارات، المفضلة، حسابي
  - قائمة منسدلة للغة والعملة
  - رابط تسجيل الدخول/التسجيل

- **شريط البحث الرئيسي**
  - حقل البحث الكبير والبارز
  - قائمة منسدلة لاختيار الفئة
  - قائمة منسدلة لاختيار الموقع
  - زر البحث الرئيسي
  - رابط "بحث متقدم"

- **القسم الرئيسي (Hero Section)**
  - بانر دوار (Carousel) للعروض المميزة
  - أزرار سريعة للفئات الأكثر شعبية
  - إحصائيات سريعة (عدد الإعلانات، المستخدمين، إلخ)

- **قسم الفئات**
  - عرض شبكي للفئات الرئيسية
  - كل فئة مع أيقونة واسم
  - عدد الإعلانات في كل فئة
  - رابط للفئات الفرعية

- **قسم الإعلانات المميزة**
  - عرض شبكي للإعلانات المميزة
  - كل إعلان في بطاقة جذابة
  - شارة "مميز" واضحة
  - معلومات أساسية (السعر، الموقع، التاريخ)

- **قسم الإعلانات الحديثة**
  - عرض شبكي للإعلانات الجديدة
  - بطاقات بسيطة مع معلومات أساسية
  - شارة "جديد" للإعلانات الحديثة

- **التذييل (Footer)**
  - روابط سريعة (عن الموقع، الشروط والأحكام، إلخ)
  - روابط الشبكات الاجتماعية
  - معلومات الاتصال
  - روابط التطبيقات

#### 1.1.2 صفحة البحث
**العناصر الرئيسية:**
- **شريط البحث العلوي**
  - حقل البحث مع النص الحالي
  - فلاتر سريعة (السعر، الموقع، النوع)
  - زر "حفظ البحث"

- **القسم الجانبي للفلاتر**
  - فلاتر حسب الفئة
  - فلاتر حسب السعر (نطاق)
  - فلاتر حسب الموقع (دولة، مدينة، منطقة)
  - فلاتر حسب الحالة (جديد، مستعمل)
  - فلاتر حسب البائع (فرد، شركة)
  - فلاتر حسب التاريخ
  - فلاتر مخصصة حسب الفئة

- **قسم النتائج**
  - عرض شبكي أو قائمة للإعلانات
  - خيارات الترتيب (الأحدث، الأقل سعراً، الأكثر سعراً)
  - شارات للإعلانات المميزة
  - معلومات أساسية لكل إعلان

#### 1.1.3 صفحة تفاصيل الإعلان
**العناصر الرئيسية:**
- **معرض الصور**
  - صورة رئيسية كبيرة
  - معرض صور مصغر
  - إمكانية تكبير الصور
  - عداد الصور

- **معلومات الإعلان**
  - عنوان الإعلان
  - السعر بارز
  - الموقع على الخريطة
  - التاريخ والوقت
  - المشاهدات
  - الشارات (مميز، جديد، موثق)

- **تفاصيل البائع**
  - اسم البائع
  - صورة الملف الشخصي
  - تقييم البائع
  - عدد الإعلانات
  - تاريخ الانضمام
  - شارة "موثق" إذا كان موثقاً

- **خيارات التواصل**
  - زر "اتصال" مباشر
  - زر "محادثة" داخل التطبيق
  - زر "إرسال عرض"
  - زر "مشاركة"
  - زر "إضافة للمفضلة"

- **أقسام إضافية**
  - وصف تفصيلي
  - مواصفات فنية
  - موقع على الخريطة
  - إعلانات مشابهة
  - إعلانات أخرى من نفس البائع

#### 1.1.4 صفحة نشر الإعلان
**الخطوات:**
1. **اختيار الفئة**
   - عرض شجري للفئات
   - بحث سريع للفئات
   - اقتراحات للفئات المشابهة

2. **إدخال التفاصيل**
   - عنوان الإعلان
   - وصف تفصيلي
   - السعر
   - الحالة (جديد، مستعمل)
   - المواصفات حسب الفئة

3. **رفع الصور**
   - رفع صور متعددة
   - إمكانية إعادة ترتيب الصور
   - إضافة وصف للصور
   - رفع فيديو اختياري

4. **تحديد الموقع**
   - اختيار المدينة
   - تحديد الموقع على الخريطة
   - إضافة وصف للموقع

5. **معلومات التواصل**
   - رقم الهاتف
   - البريد الإلكتروني
   - تفضيلات التواصل

6. **مراجعة ونشر**
   - مراجعة جميع المعلومات
   - اختيار نوع الإعلان (مجاني/مميز)
   - نشر الإعلان

### 1.2 نظام البحث والفلترة

#### 1.2.1 البحث الأساسي
- **شريط البحث الرئيسي**
  - حقل نصي كبير
  - اقتراحات تلقائية

#### 1.2.2 البحث المتقدم

  - فلاتر حسب المواصفات الدقيقة
  - فلاتر حسب البائع
  - فلاتر حسب التاريخ
  - فلاتر حسب المشاهدات

- **خيارات الترتيب**
  - الأحدث
  - الأقل سعراً
  - الأكثر سعراً
  - الأكثر مشاهدة
  - الأقرب لك

#### 1.2.3 حفظ البحث
- **حفظ عمليات البحث**
  - حفظ الفلاتر المفضلة
  - إشعارات عند ظهور إعلانات جديدة
  - مشاركة عمليات البحث المحفوظة

### 1.3 نظام الإعلانات

#### 1.3.1 أنواع الإعلانات
- **الإعلانات المجانية**
  - مدة محدودة (7 أيام)
  - عدد محدود من الصور (5)
  - ظهور في النتائج العادية

- **الإعلانات المميزة**
  - مدة أطول (30 يوم)
  - عدد أكبر من الصور (20)
  - ظهور في المقدمة
  - شارة "مميز"
  - إحصائيات متقدمة

- **الإعلانات العاجلة**
  - مدة قصيرة (3 أيام)
  - ظهور في المقدمة
  - شارة "عاجل"
  - إحصائيات فورية

#### 1.3.2 خطوات نشر الإعلان
1. اختيار الفئة والفئة الفرعية
2. إدخال التفاصيل والمواصفات
3. رفع الصور والفيديو
4. تحديد الموقع على الخريطة
5. تحديد طريقة التواصل
6. اختيار نوع الإعلان
7. مراجعة ونشر

#### 1.3.3 إدارة الإعلانات
- **لوحة تحكم الإعلانات**
  - عرض جميع الإعلانات
  - تعديل الإعلانات
  - حذف الإعلانات
  - تجديد الإعلانات
  - ترقية الإعلانات

- **إحصائيات الإعلانات**
  - عدد المشاهدات
  - عدد النقرات
  - عدد الاستفسارات
  - معدل التفاعل

### 1.4 نظام التواصل

#### 1.4.1 خيارات التواصل
- **الاتصال المباشر**

  - عرض رقم الهاتف

### 6.2 قاعدة البيانات

#### 6.2.1 الجداول الأساسية

**جدول المستخدمين (users)**
```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    email VARCHAR(255) UNIQUE NOT NULL,
    phone VARCHAR(20) UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    first_name VARCHAR(100),
    last_name VARCHAR(100),
    avatar_url TEXT,
    is_verified BOOLEAN DEFAULT FALSE,
    is_active BOOLEAN DEFAULT TRUE,
    rating DECIMAL(3,2) DEFAULT 0,
    total_reviews INTEGER DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**جدول الإعلانات (ads)**
```sql
CREATE TABLE ads (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    category_id INTEGER REFERENCES categories(id),
    subcategory_id INTEGER REFERENCES subcategories(id),
    title VARCHAR(255) NOT NULL,
    description TEXT,
    price DECIMAL(10,2),
    currency VARCHAR(3) DEFAULT 'USD',
    location_id INTEGER REFERENCES locations(id),
    status VARCHAR(20) DEFAULT 'active',
    is_featured BOOLEAN DEFAULT FALSE,
    is_urgent BOOLEAN DEFAULT FALSE,
    views INTEGER DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    expires_at TIMESTAMP
);
```

**جدول الصور (ad_images)**
```sql
CREATE TABLE ad_images (
    id SERIAL PRIMARY KEY,
    ad_id INTEGER REFERENCES ads(id),
    image_url TEXT NOT NULL,
    is_primary BOOLEAN DEFAULT FALSE,
    order_index INTEGER DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**جدول الفئات (categories)**
```sql
CREATE TABLE categories (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    name_ar VARCHAR(100) NOT NULL,
    icon VARCHAR(50),
    color_hex VARCHAR(7),
    order_index INTEGER DEFAULT 0,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**جدول الفئات الفرعية (subcategories)**
```sql
CREATE TABLE subcategories (
    id SERIAL PRIMARY KEY,
    category_id INTEGER REFERENCES categories(id),
    name VARCHAR(100) NOT NULL,
    name_ar VARCHAR(100) NOT NULL,
    icon VARCHAR(50),
    order_index INTEGER DEFAULT 0,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**جدول المواقع (locations)**
```sql
CREATE TABLE locations (
    id SERIAL PRIMARY KEY,
    parent_id INTEGER REFERENCES locations(id),
    name VARCHAR(100) NOT NULL,
    name_ar VARCHAR(100) NOT NULL,
    type VARCHAR(20) NOT NULL, -- country, city, area
    latitude DECIMAL(10,8),
    longitude DECIMAL(11,8),
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**جدول المحادثات (conversations)**
```sql
CREATE TABLE conversations (
    id SERIAL PRIMARY KEY,
    ad_id INTEGER REFERENCES ads(id),
    buyer_id INTEGER REFERENCES users(id),
    seller_id INTEGER REFERENCES users(id),
    status VARCHAR(20) DEFAULT 'active',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**جدول الرسائل (messages)**
```sql
CREATE TABLE messages (
    id SERIAL PRIMARY KEY,
    conversation_id INTEGER REFERENCES conversations(id),
    sender_id INTEGER REFERENCES users(id),
    message TEXT NOT NULL,
    is_read BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**جدول التقييمات (reviews)**
```sql
CREATE TABLE reviews (
    id SERIAL PRIMARY KEY,
    ad_id INTEGER REFERENCES ads(id),
    reviewer_id INTEGER REFERENCES users(id),
    reviewed_id INTEGER REFERENCES users(id),
    rating INTEGER NOT NULL CHECK (rating >= 1 AND rating <= 5),
    comment TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**جدول المفضلة (favorites)**
```sql
CREATE TABLE favorites (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    ad_id INTEGER REFERENCES ads(id),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(user_id, ad_id)
);
```

**جدول الإشعارات (notifications)**
```sql
CREATE TABLE notifications (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    type VARCHAR(50) NOT NULL,
    title VARCHAR(255) NOT NULL,
    message TEXT,
    data JSONB,
    is_read BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

- التحقق من صحة البيانات على جانب الخادم

#### 6.4.4 Rate Limiting

## الخلاصة

هذا التحليل الشامل يوفر خارطة طريق واضحة لتطوير منصة إعلانات مبوبة احترافية يمكنها منافسة أفضل المواقع في السوق. المفتاح للنجاح هو:

1. **التركيز على تجربة المستخدم**: تصميم بسيط وسهل الاستخدام
2. **نظام بحث قوي**: فلاتر متقدمة مع اقتراحات ذكية
3. **عرض إعلانات جذاب**: بطاقات واضحة مع شارات مميزة
4. **نظام تواصل فعال**: خيارات متعددة مع إشعارات فورية
5. **دعم متعدد للدفع**: طرق دفع متنوعة مع باقات مرنة
6. **أمان قوي**: حماية كاملة للبيانات والمستخدمين
7. **أداء ممتاز**: تحميل سريع وتجربة سلسة
8. **قابلية التوسع**: بنية قابلة للتوسع مع نمو المنصة

باتباع هذا التحليل وخارطة الطريق، يمكن تطوير منصة إعلانات مبوبة احترافية تنافس أفضل المواقع في السوق.
